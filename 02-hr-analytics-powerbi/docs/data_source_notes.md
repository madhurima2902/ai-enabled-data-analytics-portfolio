# Data Source Notes

## Source

This project uses a fictional HR analytics dataset from a DataCamp Power BI case study.

The dashboard was redesigned and documented as a portfolio project to demonstrate Power BI report development, data modeling, DAX measures, HR KPI analysis, and AI-assisted dashboard development.

## Important Note
The documentation focuses on:

- Business problem framing
- Dashboard design
- Power BI model explanation
- DAX logic
- Insights and recommendations
- Interview preparation

## Dataset Files

Original case study files included:

- Education level data
- Employee data
- Performance rating data
- Rating level lookup data
- Satisfaction level lookup data
- Date table DAX script

## Data Privacy

The dataset is fictional and does not contain real employee data.

## Limitations

- The data should not be interpreted as representing a real company.
- Attrition patterns are analytical observations, not causal proof.
- Salary comparisons should be controlled by role, tenure, department, location, and performance before drawing compensation equity conclusions.
- Some employee review timelines required validation logic to prevent misleading pre-hire review analysis.
=======
This dashboard uses a fictional HR analytics dataset from a DataCamp Power BI case study.

The dataset represents a technology workforce scenario with employee profile data, performance review records, education levels, satisfaction levels, rating levels, and a generated date dimension.

## Tables Used

| Table | Description |
|---|---|
| Employee | Employee profile, demographic, job, compensation, hire date, tenure, overtime, travel, and attrition details |
| PerformanceRating | Yearly performance review data by employee |
| EducationLevel | Education-level lookup table |
| RatingLevel | Performance rating lookup table |
| SatisfiedLevel | Satisfaction rating lookup table |
| DimDate | Calculated date table created in Power BI |
